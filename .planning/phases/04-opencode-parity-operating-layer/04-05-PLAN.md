<objective>
Add `opencode/plugins/memory-recall.ts` (OCP-02): before an OpenCode edit/write, surface AgentFS facts and wiki pages that specifically mention the file about to be edited — parity with the Claude `memory-recall.sh` PreToolUse hook and its high-signal/low-noise rule (D-10).

Per 04-RESEARCH.md, OpenCode's `tool.execute.before` cannot inject non-blocking `additionalContext` the way Claude does; the only confirmed channel is to `throw new Error(text)`, which surfaces the text to the model as the tool result and blocks that attempt (D-11 / Pattern 2). This plan implements that mechanism with a once-per-file guard so a matched edit is not blocked in an infinite loop.

Purpose: Turn "the model might search memory" into "the model sees the exact file's pitfalls/decisions before editing it."
Output: `opencode/plugins/memory-recall.ts`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OpenCode edit/write tool call → recall plugin | untrusted target `filePath` crosses into plugin code |
| recall plugin → local cairn-memory server | plugin runs `node <INFRA_ROOT>/mcp-memory-server/dist/index.js wakeup` |
| recall plugin → `.planning/wiki/sources/` | plugin greps wiki source files selected by the derived stem |
| recall plugin → model context | thrown Error text is surfaced to the model as the tool result |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-02 | Tampering / Information Disclosure | untrusted `filePath` driving a filesystem read outside `.planning/wiki/sources` (path traversal) | medium | mitigate | Use the file path only to derive a basename/stem grep token; confine the `find`/read root to `.planning/wiki/sources/` with `relative()`-based containment (SEC-0001), never concatenate the untrusted path into a read path |
| T-04-03 | Elevation of Privilege | INFRA_ROOT resolving to an attacker-controlled `node` target | medium | mitigate | Resolve the server path only from the install-rendered `@@INFRA_ROOT@@` (repo root computed by the sync script in 04-06), never from env/tool input |
| T-04-05 | Information Disclosure | thrown-error text echoing matched memory/wiki content into model context | low | accept | Content is the project's own memory/wiki, not secrets; this phase adds a delivery channel, not a new data source |
| T-04-09 | Denial of Service | repeat-throwing blocking the same edit in an infinite retry loop | medium | mitigate | Once-per-file-per-session guard: after throwing for a file, let the retry proceed unmodified so the model is never stuck |
| T-04-04 | Denial of Service | fail-open abuse (induced errors suppress recall) | low | accept | D-03 deliberately trades enforcement for availability — on any error the edit proceeds unmodified |
| T-04-SC | Tampering | `@opencode-ai/plugin` (SUS scanner verdict) | high | mitigate | Type-only, runtime-resolved import (no install); a blocking-human checkpoint verifying `npmjs.com/package/@opencode-ai/plugin` is required only if an executor introduces a `package.json` install |
</threat_model>
