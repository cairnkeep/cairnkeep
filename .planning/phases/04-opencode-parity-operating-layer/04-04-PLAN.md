## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OpenCode session → capture plugin | untrusted session/message content crosses into plugin code |
| capture plugin → local cairn-memory server | plugin pipes session text into `node <INFRA_ROOT>/mcp-memory-server/dist/index.js extract` |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering (Injection) | session/message text passed to the `extract` subprocess | high | mitigate | Pass session text as `$` stdin, never interpolate it into the command string — arbitrary message content (shell metacharacters, prompt-injection payloads) cannot reach a shell parser; the extraction model receives it only as data |
| T-04-03 | Elevation of Privilege | INFRA_ROOT resolving to an attacker-controlled `node` target | medium | mitigate | Resolve the server path only from the install-rendered `@@INFRA_ROOT@@` (repo root computed by the sync script in 04-06), never from env/session input |
| T-04-08 | Denial of Service | `session.idle` firing repeatedly / for subagents over-staging past the 5-cap intent | medium | mitigate | Filter out `parentID`-bearing subsessions and dedupe per `sessionID`; enforce the 5-newest retention cap after every write |
| T-04-04 | Denial of Service | fail-open abuse / dropped async extract | low | accept | D-03 + Pitfall 3: a missing staging file is a tolerated degraded outcome; the handler never wedges a session or retry-loops |
| T-04-SC | Tampering | `@opencode-ai/plugin` / optional `@opencode-ai/sdk` (SUS scanner verdict) | high | mitigate | Type-only, runtime-resolved imports (no install); a blocking-human checkpoint verifying `npmjs.com/package/@opencode-ai/plugin` (and `/@opencode-ai/sdk`) is required only if an executor introduces a `package.json` install |
</threat_model>
