<objective>
Build the offline, fail-closed smoke harness for `context_explore` BEFORE the tool
exists (Wave 0 test infrastructure). This plan produces the Node smoke script, the
fake-binary fixtures that stand in for `token_miser` without a live FastContext
endpoint, and the `package.json` wiring that folds the guard into `test:smoke`.

The harness is written to be RED now (the `context_explore` tool is not yet
registered) and to turn GREEN once Plan 02 lands. It is the automated verification
vehicle every Plan 02 task's `<verify>` block runs against, and it discharges
ROADMAP SC-4 ("an offline smoke test exercises the not-configured and binary-missing
fail-closed paths and passes in CI").

Purpose: Satisfy the Nyquist "test scaffold first" rule and give the implementation
plan a real, offline, deterministic pass/fail gate for CTX-02's fail-closed contract
and CTX-01's Evidence -> compact-citation rendering.
Output: `smoke-explore-guard.mjs`, four fixture scripts, and the `check:explore-guard`
package script.
</objective>

Stands in for a real successful exploration so the Evidence -> compact `path:line-range` rendering (CTX-01) is offline-testable without a live model.
