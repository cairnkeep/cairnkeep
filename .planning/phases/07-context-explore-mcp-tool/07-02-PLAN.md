<objective>
Register the `context_explore` MCP tool in the existing single-file cairn-memory
server, delegating one natural-language query to the external `token_miser explore`
subprocess via the existing `runCommand` helper, parsing its `Evidence` JSON, and
failing closed on every error path. This is the thin adapter the milestone exists
for: precondition validation + subprocess plumbing + one JSON reshape into dual
text/structured output. No exploration logic is re-implemented (token_miser owns it).

This plan turns the Wave 1 offline guard (`smoke-explore-guard.mjs`) from RED to
GREEN and satisfies CTX-01 (delegate + parse + compact citations), CTX-02 (fail
closed, never silent empty-success), and CTX-03 (env-only config, no vendor default).

Purpose: Deliver the tool exactly per CONTEXT.md D-01..D-04 and the empirically
pinned CLI/JSON contract in 07-RESEARCH.md.
Output: One new `server.registerTool("context_explore", ...)` block plus a minimal,
backward-compatible `env` parameter on `runCommand`, both in
`mcp-memory-server/src/index.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> context_explore | Untrusted tool args (`query`, `repo_root`, `timeout_seconds`) enter here; validated by the zod inputSchema (V5). |
| context_explore -> token_miser subprocess | Args cross via a spawn argv array (no shell); `--repo-root` is an absolute, existence-checked path (V12). |
| token_miser -> its own config/endpoint | FastContext endpoint/model/credentials live entirely on token_miser's side; cairn-memory never sees or sets them (CTX-03 by construction). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Tampering | `query`/`repo_root` containing shell metacharacters | low | accept | Non-vector: `runCommand` calls `spawn(cmd, argv[], {})` with no `shell:true`; args reach `execve` directly, never a shell. Unchanged by this phase (07-RESEARCH.md §Security Domain, VERIFIED runCommand L406-446). V5 zod requires non-empty `query`. |
| T-07-02 | Information Disclosure | passthrough `stderr` in `{ ok:false, ..., stderr }` | low | accept | Non-vector: cairn-memory holds no FastContext secret, so it has none to leak. stderr can at most carry token_miser's own diagnostics (config lives on token_miser's side). `NO_COLOR=1` keeps that text clean (Pitfall #2). |
| T-07-03 | Tampering / Elevation of Privilege | caller-supplied `repo_root` outside any "expected" dir | low | accept | Intended documented semantics (unlike memory_read's scope confinement): the tool explores whatever repo the caller names; token_miser's own Sandbox confines READ/GLOB/GREP within that root. Same trust model as the existing `runCommand` caller (the MCP client already has FS access). |
| T-07-04 | Denial of Service | slow / hung exploration | low | mitigate | `timeout_seconds` bounded `min(10).max(600)` in the schema; `runCommand` SIGTERMs on timeout and the handler returns `{ ok:false, error:"timed out", timedOut:true }` (fail-closed, never a hang-as-success). |
| T-07-05 | Spoofing / Tampering (path) | relative `repo_root` exploring the wrong tree | medium | mitigate | Resolve to an ABSOLUTE path (`resolve(expandHome(rawRoot))`) and `existsSync`-check BEFORE spawn; never pass a relative string to `--repo-root` (Pitfall #3, D-01). |
| T-07-06 | Repudiation | unreachable-but-configured endpoint -> exit 0 + empty Evidence, indistinguishable from a genuine empty result | low | accept | Residual, documented CTX-02 gap that cannot be closed without re-implementing token_miser's explorer loop (out of scope). Mitigation is transparency: surface `stats.turns`/`stats.tool_calls` in the empty-citation text so a caller can spot a suspicious `tool_calls:0` (Pitfall #1). Recorded in verification notes, not silently treated as discharged. |
| T-07-SC | Tampering | npm/pip/cargo installs | low | accept | Zero new packages this phase (07-RESEARCH.md §Package Legitimacy Audit — N/A); no supply-chain surface added. |

No threat reaches `high`; the ASVS L1 gate (block on `high`) is not tripped — every item is a non-defect by construction or is mitigated in-handler. T-07-06 is the one accepted residual gap in CTX-02, documented per the "thin adapter" phase boundary.
