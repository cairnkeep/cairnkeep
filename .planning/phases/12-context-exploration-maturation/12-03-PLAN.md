|----------|-------------|
| user prompt → hook → explore CLI | The submitted prompt (possibly containing pasted untrusted text) becomes the explore query; passed as a single argv to node, never a shell string |
| hook process → Claude Code lifecycle | The hook runs inside Claude Code's UserPromptSubmit budget; a runaway must not hang the session |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-07 | Denial of Service | hook blocking Claude Code | high | mitigate | Explicit short `timeout 20` on the CLI call + explicit per-hook timeout in settings.json (Pitfall 1); double opt-in (D-07) limits how often it fires; fail-open exit 0 |
| T-12-08 | Tampering | prompt text reaching subprocess | medium | mitigate | Prompt passed as a single argv element to `node ... explore "$prompt"`; server-side runCommand already uses argv-array spawn (V5) — no shell interpolation |
| T-12-09 | Denial of Service | cache growth from high-frequency auto-invoke | medium | mitigate | Plan 01's ~200-entry oldest-first prune + the D-07 opt-in gate bound write frequency |

No package installs this phase — no T-12-SC supply-chain checkpoint required.
