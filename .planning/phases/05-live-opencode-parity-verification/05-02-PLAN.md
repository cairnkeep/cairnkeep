<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| harness shell → `opencode run` subprocess | CAIRN_LLM_* exported in this shell (05-01) reach the plugins' direct `node <server>` shell-outs — the capture/wakeup path |
| OpenCode → `cairn-memory` stdio MCP subprocess | remember/recall go through cairn-memory_memory_* tool calls; server name must be exactly cairn-memory |
| plugin → `node <server> wakeup\|extract` shell-out | inherits OpenCode's process env, not the MCP environment block |
| local model endpoint (loopback) → harness assertions | untrusted model output is only asserted (canary FOUND/NOT-FOUND), never executed |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Information Disclosure | committed harness stages + assertions | medium | mitigate | Canaries are runtime-generated synthetic tokens; assertions reference the runtime value, never a committed literal; no private host in the script (DEC-no-private-references) |
| T-05-02 | Tampering | scratch config + real config | high | mitigate | main()/cleanup asserts real ~/.config/opencode and ~/.claude unmodified after every full run |
| T-05-03 | Spoofing (false positive) | canary provenance | medium | mitigate | Negative control per stage (unseeded project → NOT-FOUND) proves a surfaced canary came from injected memory, not model training/guessing (D-04) |
| T-05-04 | Denial of Service | headless permission prompts | low | mitigate | --auto on every tool-call run; permissive permission block in the scratch config (Pitfall 4) |
| T-05-SC | Tampering (supply chain) | package installs | low | accept | No package installs; exercises already-audited Phase-4 code + the built v1.0-validated server binary |
</threat_model>
