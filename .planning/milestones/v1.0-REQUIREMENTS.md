**Defined:** 2026-07-03 (ingested from existing planning docs; IDs preserved from .planning/intel/requirements.md)

## v1 Requirements (Milestone: OSS core → parity)

Each requirement maps to exactly one roadmap phase.

### Memory Server

- [x] **REQ-memory-mcp-server**: The `cairn-memory` MCP server registers and responds as an MCP server, with remember/recall operations working end-to-end — ✓ validated (v1.0)

### Operating Layer

- [x] **REQ-operating-layer**: Commands, agents, and hooks for memory, wiki, security, and review workflows work end-to-end against the `cairn-memory` MCP across harnesses (Claude Code, OpenCode, ...), including memory hooks (wakeup/capture/review) round-tripping — ✓ validated (v1.0); Claude Code is the complete verified path, OpenCode wakeup ordering documented

### CLI & Bootstrap

- [x] **REQ-cli-bootstrap**: A fresh `cairn bootstrap` yields a working same-as-before workflow (launchers + derived-knowledge layer) — ✓ validated (v1.0) into a scratch `--live-root`

### Core Neutrality

- [x] **REQ-provider-neutral-core**: Provider-neutral core — git host selected by one config setting, collaboration commands resolve every git-host operation through a per-provider operation→tool map, no hardcoded vendor endpoints anywhere in the core (delivered in Phase 1)

### Parity & Hygiene

- [x] **REQ-feature-parity**: Feature parity with the originating private workflow — operating guide in the OSS docs, fresh bootstrap works same-as-before, baseline tagged at sign-off — ✓ validated (v1.0); baseline `v1.0.0`
- [x] **REQ-oss-hygiene**: Clean open-source hygiene — Apache-2.0 license present, CI in place, no secrets or attribution noise in any artifact — ✓ validated (v1.0)
