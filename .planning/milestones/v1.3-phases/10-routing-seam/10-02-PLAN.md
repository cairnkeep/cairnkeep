<objective>
Seal the routing seam for external consumption: (1) a re-runnable verify script
that starts the **real** token_miser binary and proves it answers `GET /health`
(D-06 verify-by-execution — this repo's bar, above docs-only per D-10), and
(2) the RT-02 seam-contract documentation in `docs/operating.md` so a future
private overlay can drive routing using the doc alone (SC #3).

Per RESEARCH.md: `/health` reachability proves the **seam's target is a real,
live binary** — it does not prove a tier decision (the tier is never surfaced
over HTTP; `/health` never calls `Router::classify`). The doc must state this
distinction explicitly (Pitfall 3/4) or SC #3 breaks. The full
`/v1/chat/completions` round-trip needs a live tier backend absent on this
machine, so it is an explicitly optional `--full` operator-gated stretch, never
the minimum (D-06, D-07 defer live routing).

Purpose: make the frozen seam real (proven against the actual binary) and
externally driveable (documented to the no-source-reading bar).
Output: `scripts/verify-routing-seam.sh`, the `CAIRN_ROUTE_ENDPOINT` doc row,
and the Routing seam contract subsection.
</objective>
