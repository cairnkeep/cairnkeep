<objective>
Add `route_check`, a thin fetch-based MCP tool in the cairn-memory server that
delegates to an already-running external token-miser routing proxy by probing
its `GET /health` endpoint — and prove it end-to-end with an offline MCP
round-trip guard. This is the RT-01 delegate.

Per RESEARCH.md (HIGH confidence, verified against token-miser source + live
binary): token-miser routing is **proxy-only HTTP**, so the seam is the D-03
env-var endpoint *reference* (`CAIRN_ROUTE_ENDPOINT`), consumed via `fetch()`
(mirroring `extractMemoryCandidates`), **not** a `runCommand`/argv subprocess
(D-01/D-02 resolved FALSE — no `token_miser route` subcommand exists). The tool
borrows `context_explore`'s registration + dual-output + fail-closed tier split
(D-04, D-09), the env-only no-defaults config idiom (D-05, D-08), and freezes
the seam via pinning assertions (D-10). Zero new npm packages.

Purpose: give a future overlay one driveable, independently-verifiable entry
point to token-miser's routing surface while the core hosts no proxy, endpoint
list, model list, or tier config (RT-01, SC #1).
Output: `route_check` tool, `smoke-route-guard.mjs`, `check:route-guard` wired
into `test:smoke`.
</objective>
