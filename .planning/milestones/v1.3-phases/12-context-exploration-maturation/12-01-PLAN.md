## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller query → git/subprocess | The `query` string (later from an untrusted prompt) reaches git and the token_miser binary; must stay argv-array, never a shell string |
| explored repo_root → filesystem cache | Cache files are written based on a caller-supplied repo_root; must never be written inside that repo |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-01 | Tampering | explore-cache.ts git reads | low | mitigate | `execFileSync("git", [argv])` — argv array, no shell interpolation of query/repoRoot (V5); reuses the repo's established argv-spawn discipline |
| T-12-02 | Denial of Service | cache dir growth | medium | mitigate | Oldest-first prune at write time, ~200-entry cap (D-11, RESEARCH V12) |
| T-12-03 | Information Disclosure | cache file location | low | mitigate | Cache written only under `${XDG_CACHE_HOME:-~/.cache}/cairn/explore/`, never inside the explored repo (D-11) — prevents leaking cached evidence into an arbitrary repo tree |

No package installs this phase (RESEARCH: zero new npm/pip/cargo packages) — no T-12-SC supply-chain checkpoint required.
